# model.joblib = rank-fused ensemble:
#   stack = StackingClassifier[LGBMClassifier, XGBClassifier, RandomForestClassifier] -> LogisticRegression
#   mono  = VotingClassifier[3x LGBMClassifier]
#   mlp   = Pipeline[StandardScaler, PCA, sklearn MLPClassifier]   (NO torch)
# so the serving venv MUST have lightgbm + xgboost + scikit-learn.
#
# CRITICAL PINS:
#  * xgboost MUST be 2.x — the pickled XGBClassifier buffer fails to load on
#    xgboost 3.x ("XGBoostError: input stream corrupted"). Verified on 2.1.4.
#  * scikit-learn 1.9.0 — the artifact (StackingClassifier / trees / MLP / PCA)
#    was pickled with 1.9.0; match it exactly or unpickling is not guaranteed.
#  * Missing lightgbm/xgboost => the ensemble fails to load => every request
#    routes to the [0.5]*n exception fallback and scores zero reward.
lightgbm>=4.0
xgboost>=2.0,<3.0
scikit-learn==1.9.0
joblib>=1.3
numpy>=1.24

# Bittensor MUST stay on 9.x for this codebase. In bittensor 10.x,
# bt.Config(parser=...) skips CLI parsing entirely unless
# BT_NO_PARSE_CLI_ARGS=false (core/settings.py:no_parse_cli defaults ON),
# so --wallet/--netuid/--axon.port are ignored and config.neuron is None ->
# "AttributeError: 'NoneType' object has no attribute 'name'" at startup.
# 9.6.0 is what the subnet code targets; 9.12.2 for Python 3.14.
bittensor==9.6.0; python_version < "3.14"
bittensor==9.12.2; python_version >= "3.14"
# bittensor 9.x under-constrains its companions, so pip pairs 9.6.0 with
# 10.x-era packages that break it at import (bittensor-drand 2.0 removed
# get_encrypted_commit; async-substrate-interface 2.x is the 10.x line).
bittensor-drand>=1.0.0,<2.0.0
async-substrate-interface>=1.5.6,<2.0.0
bittensor-wallet>=3.0.8,<5.0
